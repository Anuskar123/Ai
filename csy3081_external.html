1. Introduction to Artificial Intelligence & History (`[00:00 - 27:12]`)

Artificial Intelligence (AI) is the overarching engineering discipline focused on creating computational agents capable of perceiving environments, synthesizing data, and automating complex cognitive decisions once exclusive to human intelligence (`[08:46]`). The course traces AI from early **Symbolic Logic & Turing Test formulations (`[02:27]`)** to modern statistical machine learning.

ðŸ“Œ The AI vs. Machine Learning vs. Deep Learning Hierarchy (`[08:46]`):
  • Artificial Intelligence (AI): Broad umbrella encompassing rule-based expert systems, logic algorithms, and self-learning systems.
  • Machine Learning (ML): A specialized subset of AI where algorithms automatically learn patterns and decision boundaries from training data rather than relying on explicit hand-coded rules (`[27:12]`).
  • Deep Learning (DL): A subset of ML utilizing multi-layered artificial neural networks designed to extract hierarchical representations from raw data (e.g., computer vision pixel hierarchies).

Why Python is the Dominant Language for AI & ML (`[20:24]`)

While R, C++, and Java play roles in statistical modeling and high-performance execution, **Python** dominates the CSY3081 curriculum (`[20:24]`) due to three architectural advantages: (1) Rich scientific ecosystem (`numpy`, `pandas`, `scikit-learn`, `matplotlib`), (2) Clean, expressive syntax enabling rapid prototyping, and (3) Seamless C/C++ bindings via NumPy array buffers that execute complex vector algebra at compiled speeds.

3. Linear & Logistic (Softmax) Regression (`[1:04:49 - 1:35:36]`)

Linear Regression (`[1:05:17]`) predicts continuous target variables $y \in \mathbb{R}$ by fitting a linear hyperplane through the feature space. The model minimizes the Mean Squared Error (MSE) cost function either via iterative Gradient Descent or the analytical Normal Equation (`[1:11:20]`):

$$J(\theta) = \frac{1}{2m} \sum_{i=1}^{m} \left( h_\theta(x^{(i)}) - y^{(i)} \right)^2 \quad \text{where } h_\theta(x) = \theta^T x$$

Logistic & Softmax Multiclass Regression (`[1:26:36]`)

For binary classification ($y \in \{0, 1\}$), Logistic Regression applies the non-linear **Sigmoid function $\sigma(z) = \frac{1}{1 + e^{-z}}$** to map linear predictions into $(0, 1)$ probabilities. For **Multiclass Classification ($K > 2$ classes)**, such as our `logistic softmax.docx` activity on the Iris dataset (`[1:26:36]`), the model uses the **Softmax Function**:

$$P(y = k \mid x; \theta) = \frac{e^{\theta_k^T x}}{\sum_{j=1}^{K} e^{\theta_j^T x}}$$
Scikit-Learn Implementation: Softmax Regression (`logistic softmax.docx`)

4. Decision Trees & Random Forest Bagging Ensembles (`[1:35:36 - 2:07:31]`)

Decision Trees (`[1:35:36]`) are non-parametric supervised learning models (`Handson on Decision Tree.pdf`) that recursively partition the feature space using axis-aligned splitting rules. Unlike distance-based algorithms (`KNN` or `SVM`), **Decision Trees require ZERO feature scaling** (`StandardScaler` is unnecessary).

Impurity & Splitting Criteria (`Gini vs. Entropy`)

To evaluate the quality of a split at any node $m$, classification trees minimize impurity across child nodes using either **Gini Impurity** (default in `DecisionTreeClassifier`) or **Information Gain (Entropy)**:

$$\text{Gini}(m) = 1 - \sum_{k=1}^{K} p_{mk}^2 \quad \text{vs.} \quad \text{Entropy}(m) = - \sum_{k=1}^{K} p_{mk} \log_2(p_{mk})$$
ðŸ’¡ Visualizing 1D Regression Trees (`[1:48:00]`):

As explored in `Handson on Decision Tree.pdf (Activity 2)`, when `DecisionTreeRegressor` is plotted over a continuous input feature (`X_plot`), the resulting prediction curve forms a **Step Function** (horizontal flat segments). Because every leaf node outputs the average target value of all training samples within that partition, a single Decision Tree **cannot extrapolate outside the bounds of its training data**.

Random Forest Ensemble & Bagging (`[1:55:18]`)

Single deep decision trees (`max_depth=None`) suffer from **High Variance (Overfitting)**. **Random Forest (`[1:55:18]`)** mitigates this by constructing an ensemble of $M$ independent decision trees (`n_estimators`) using two core randomization techniques:

  1. Bootstrap Aggregation (Bagging): Each tree is trained on a random bootstrap sample ($N$ instances drawn with replacement from the training set).
  2. Random Subspace Method: At each node split, only a random subset of features ($m \approx \sqrt{p}$) is evaluated, forcing individual trees to be statistically uncorrelated. Final prediction is made via majority voting (`classifier`) or averaging (`regressor`).

5. Naive Bayes, K-Nearest Neighbours & Support Vector Machines (`[2:07:31 - 2:42:36]`)

Naive Bayes Classifier (`[2:07:31]`)

Generative probabilistic classifier based on **Bayes' Theorem**. It makes the "naive" conditional independence assumption that all features $x_1, x_2, \dots, x_n$ are statistically independent given the class label $y$ (`[2:07:31]`):

$$P(y \mid x_1, \dots, x_n) = \frac{P(y) \prod_{i=1}^{n} P(x_i \mid y)}{P(x_1, \dots, x_n)}$$

K-Nearest Neighbours (KNN) (`[2:14:37]`)

KNN (`[2:14:37]`) is a non-parametric instance-based ("lazy") learning algorithm. To classify a test instance $x_{\text{test}}$, it computes the geometric distance across all $N$ training points, identifies the $K$ closest neighbors, and assigns the majority class label. Because distance calculation directly drives classifications, **Feature Scaling (`StandardScaler`) is strictly mandatory**:

$$d(x, y) = \sqrt{\sum_{i=1}^{n} (x_i - y_i)^2} \quad \text{(Euclidean Metric } L_2 \text{)}$$

Support Vector Machines (SVM) & Kernel Trick (`[2:20:31]`)

SVM (`[2:20:31]`) identifies the **Maximum Margin Hyperplane** that maximally separates binary classes ($w^T x + b = 0$). When data is non-linearly separable in its native input space, SVM applies the **Kernel Trick (`kernel='rbf'`)**, which computes dot products in a high-dimensional feature space $\Phi(x)$ where linear separation becomes trivial without explicitly projecting every vector.

6. K-Means Clustering, Market Basket & Reinforcement Learning (`[2:42:36 - 3:33:08]`)

K-Means Clustering (`[2:42:45]`) partitions $N$ unlabeled data vectors into $K$ mutually exclusive clusters (`tutorial unsupervised learning (1).docx`). The algorithm alternates between assigning points to the nearest centroid and re-computing centroid coordinates (`kmeans.cluster_centers_`) to minimize the Within-Cluster Sum of Squares (`Inertia`):

$$\text{Inertia} = \sum_{k=1}^{K} \sum_{x \in S_k} \left\| x - \mu_k \right\|^2$$
Scikit-Learn Implementation: K-Means (`tutorial unsupervised learning`) 
from sklearn.cluster import KMeans
from sklearn.metrics import adjusted_rand_score

# Initialize KMeans with 4 clusters (`[2:42:45]`)
kmeans = KMeans(n_clusters=4, init='k-means++', n_init=10, random_state=42)
kmeans.fit(X_scaled)

# Retrieve cluster IDs and final centroid coordinates
cluster_labels = kmeans.labels_
centroids = kmeans.cluster_centers_ # [4 clusters, n_features]

# Evaluate against true labels using Adjusted Rand Index (ARI) (`[2:53:00]`)
ari = adjusted_rand_score(y_true, cluster_labels)
print(f"KMeans ARI Score: {ari:.4f} (Ranges -1.0 to +1.0)")

Apriori Market Basket Association Rules (`[2:54:30]`)

In retail transaction mining (`Online_Retail.xlsx`), Apriori discovers co-occurrence rules $A \Rightarrow B$ evaluated across three core statistical metrics (`[2:54:30]`):

  • Support $(A \Rightarrow B)$: $\frac{\text{Transactions containing } A \text{ and } B}{\text{Total Transactions } N}$
  • Confidence $(A \Rightarrow B)$: $\frac{\text{Support}(A \cup B)}{\text{Support}(A)}$ (Conditional probability $P(B|A)$)
  • Lift $(A \Rightarrow B)$: $\frac{\text{Confidence}(A \Rightarrow B)}{\text{Support}(B)}$. If **Lift $= 1.0$**, items $A$ and $B$ are **completely independent** (`tutorial unsupervised learning`). If **Lift $> 1.0$**, they positively attract co-purchase.

Reinforcement Learning & Exploration vs. Exploitation (`[2:56:40]`)

Unlike supervised algorithms that learn from static labeled pairs, an **RL Agent (`[2:56:40]`)** interacts dynamically with an environment. At each step $t$, it observes state $S_t$, executes action $A_t$, receives reward $R_{t+1}$, and updates value estimation tables (like Q-Learning). A core requirement is balancing **Exploration** (trying novel actions to discover better states) against **Exploitation** (choosing the action with the highest currently known reward).

7. The 5 Core Limitations of Machine Learning (`[3:33:08]`)

The course concludes (`[3:33:08]`) by outlining the five critical boundaries and ethical challenges that every CSY3081 engineer must consider when deploying real-world AI systems:

1. Data Dependency ("Garbage In, Garbage Out")

ML models possess zero intrinsic common sense; their accuracy is strictly upper-bounded by the quality, completeness, and cleanliness of the training data (`[3:33:08]`).

2. Algorithmic Bias & Ethical Fairness

If historical training datasets reflect societal prejudices or demographic imbalances, algorithms will encode, scale, and amplify discriminatory outcomes across critical domains.

3. Black-Box Interpretability Deficits

While linear regression offers clear $\theta$ weights, complex models like Random Forests (`[1:55:18]`) and Deep Neural Networks act as unexplainable black boxes, complicating regulatory auditing.

4. Susceptibility to Overfitting & Brittleness

Without proper cross-validation (`[37:26]`) and regularization (`max_depth`), models memorize training noise and fail catastrophically when encountering out-of-distribution real-world inputs.

5. High Computing Cost & Environmental Footprint

Training modern ensemble architectures (`[1:55:18]`) requires massive GPU/CPU computational cycles, leading to significant financial expense and carbon emission impacts.
